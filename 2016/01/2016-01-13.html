An Introduction to CSS’s @supports Rule (Feature Queries)

Frontend

一般两种法来解决浏览器的最新技术的支持：优雅降级和渐进增强。
优雅降级利用先进技术来设计复杂的用户体验和功能。降低用户的浏览器能力仍然能够访问该网站,但会降低水平的功能和浏览体验。
使用渐进增强,开发人员建立一个基线设计大多数浏览器可以支持的用户体验。他们的应用程序提供内置的浏览器检测功能,它们使用提供更高级的功能和更丰富的浏览体验。
最广泛采用的工具在一个渐进增强方法是Modernizr JavaScript库。
Modernizr编程检查浏览器是否支持下一代网络技术和相应的返回真或假。有了这些知识,您可以利用新特性支持的浏览器,和还有一个浏览器迎合老的或不相容的可靠手段。
这听起来一样好,甚至更好的已经酝酿了一段时间。您可以执行功能检测带有@supports规则查询使用本机的CSS特性。
在这篇文章中我将深入了解@supports及其相关的JavaScript API。
eg:

@supports (mix-blend-mode: overlay) {

  .example {
    mix-blend-mode: overlay;
  }

}

浏览器应用不同的样式,没有mix-blend-mode支持,您将使用这个语法: 

@supports not(mix-blend-mode: overlay) {

  .example {
    /* alternative styles here */
  }

}

多个所需条件: 

@supports (property1: value1) and (property2: value2) {
  element {
    property1: value1;
    property2: value2;
  }
}

通过使用分隔或关键字,您可以测试多个替代功能的风格。 

@supports (property1: value1) or (-webkit-property1: value1) {
  element {
    -webkit-property1: value1;
    property1: value1;
  }
}

你也可以结合使用or或者and: 

@supports ((property1: value1) or 
      (-webkit-property1: value1)) and 
      (property2: value2) {
  element {
    -webkit-property1: value1;
    property1: value1;
    property2: value2;
  }
}
