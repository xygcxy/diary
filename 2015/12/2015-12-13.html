关于ID的一些事

Frontend

关于全局变量和ID属性JavaScript中：每次添加一个id属性在HTML页面中的元素，将成为一个全局变量（即该窗口对象的属性）在你的脚本访问的页面上。例如，你有以下HTML：

<body id="myBody" class="home">
  <div id="one">This is div one</div>
  <div id="two"><p>This is div two</p></div>
  <div id="three"></div>
  <div id="four" data-name="fourDiv"></div>
  <div id="innerHeight"></div>
</body>

你可以访问任何的HTML元素，通过简单地使用id属性的值：

console.log(myBody.className); // "home"
console.log(one.textContent); // "This is div one"
console.log(two.innerHTML); // "<p>This is div two</p>"
console.log(three.id); // "three"
console.log(four.dataset.name); // "fourDiv"
console.log(innerHeight); // already global

注意，脚本不能使用getelementbyid()，queryselector()访问元素。指的是直接的标识属性。所有这些都可以写为“窗口”，如果你想更明确。 但是，正如你会注意到最后一个日志。如果一个全局变量已经存在于选定的标识值，它将不工作，但将代替原来的全局变量。在这种情况下，最后一个例子引用“innerheight”这已经是窗口对象，所以它只是输出窗口的内部高度。其他的例子包括 history, location, navigator, innerWidth, and much more.
