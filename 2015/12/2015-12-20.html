Understanding the CSS Modules Methodology

Frontend

当我们说CSS,我要说的最后一个重大转变是我们写CSS(在工具方面)CSS处理器的到来,大多指Sass,最著名的一个。还有PostCSS,本质上稍微不同,但最终都是作用相似。 现在,一个新的CSS模块出现了。在这篇文章中,我想介绍一下这种技术,告诉你为什么它有一些优点,以及如何开始。 一个CSS模块是一个CSS文件,所有类名和默认动画名称局限在本地。

CSS模块:一种CSS组件方法和避免全局命名空间。不用再努力找到一个好的方法命名您的组件！ 如何工作： CSS模块需要管道在构建步骤中。需要插件webpack或Browserify。它的工作原理是这样的:当你导入一个CSS文件在一个JavaScript模块，CSS模块将定义一个对象映射的类名文件动态范围/名称空间类名称,可以使用JavaScript字符串。请允许我用一个例子说明: 下面是一个非常简单的CSS文件。.base类不需要独特的项目,因为它不是实际的类名称。它只是一种别名在样式表中使用JavaScript模块。 

.base {
  color: deeppink;
  max-width: 42em;
  margin: 0 auto;
}

这里在一个JavaScript组件中使用: 

import styles from './styles.css';

element.innerHTML = `<div class="${styles.base}">
  CSS Modules are fun.
</div>`;

最终,这将产生这样的(当使用它通过webpack默认设置): <div class="_20WEds96_Ee1ra54-24ePy">CSS Modules are fun.</div> 类名可以配置为生成间短名称,或遵循一个特定的模式。重点是动态生成的,独特的。
