An Introduction to Hardware Acceleration with CSS Animations

Frontend

1.硬件加速： 我们做一个动画，是一个方形的圈。最简单的方法是调整左、上属性。我们用CSS来作：

.ball-running {
  animation: run-around 4s infinite;
}

@keyframes run-around {
  0%: {
    top: 0;
    left: 0;
  }

  25% {
    top: 0;
    left: 200px;
  }

  50% {
    top: 200px;
    left: 200px;
  }

  75% {
    top: 200px;
    left: 0;
  }
}

运行以后看到其帧数很难达到60帧。我们需要使用translate来代替top和left：

@keyframes run-around {
  0%: {
    transform: translate(0, 0);
  }

  25% {
    transform: translate(200px, 0);
  }

  50% {
    transform: translate(200px, 200px);
  }

  75% {
    transform: translate(0, 200px);
  }
}

2.硬件加速的工作原理： 当浏览器接收到一个网页，解析构建DOM树。DOM树和CSS允许浏览器构建渲染树。渲染树由渲染对象组成，在页面上呈现的元素。每个渲染对象被分配给一个图形层。每一层都是GPU作为一个纹理。这里的诀窍是，层可以在GPU下达到不会重绘，像3D图形。 在我们的例子中，创建一个新的CSS变换的复合层，可以在GPU直接转化。 在下列情况下，它是这样做的： 3D或CSS变换角度（在我们的例子中） 对于 <video> or <canvas>元素 当使用CSS滤镜 一个元素与另一个元素中提取的复合层（例如，使用z-index）

三维和二维变换的区别在于，三维变换使浏览器预先生成一个单独的复合层。在动画的开始，一个新的复合层被创造被加载到GPU，引发重绘。那么动画是通过在GPU排序进行。当动画完成后，除去额外的复合层，结果操作另一个重绘。

3.GPU渲染元素： 只有以下元素可以引起：
transform
opacity
filter
有时可以通过强制使用硬件加速来达到流畅的动画效果。使用以上的属性替代一些属性。

4.谨慎使用硬件加速 存储 加载太多纹理的GPU可能导致内存问题。这对移动设备是非常致命的。 字体渲染 在GPU渲染影响字体反锯齿。这是因为GPU和CPU有不同的渲染机制。因此，即使在动画结束时关闭硬件加速，文本将在动画中显示模糊。
