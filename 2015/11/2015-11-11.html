The Importance of import and export

Frontend

1.CommonJS特点： 所有代码都在特定模块范围内运行。 可以给任何一个给定的模块运行，当你需要它时。 根据简单规则，模块源代码是从静态文件加载。 模块加载顺序执行。 模块出口保持不同。 全局变量不被污染。

2.循环依赖： 为了避免无限循环，需要打破这个闭环。 如果出现依赖闭环，那么外部模块在被它的传递依赖（transitive dependencies）所包含的时候可能并没有执行完成；在这种情况下，"require"返回的对象必须至少包含外部模块在调用会进入当前模块执行环境的require函数之前就已经准备好了的输出。 eg:

// wrap.js
exports.deferred = function (fn) {
  return function wrapper() {
    setTimeout(fn, 0);
  };
};

var log = require("./log").log;

exports.logged = function (fn) {
  return function wrapper(...args) {
    log("calling the function");
    return fn.apply(this, args);
  };
};// log.js
var wrap = require("./wrap");

exports.log = wrap.deferred(
  function (...messages) {...}
);


// main.js
var wrap = require("./wrap");
require("http").createServer(
  wrap.logged((req, res) => {...})
).listen(8080);

3.exports VS module.exports Detail explaination exports 是指向的 module.exports 的引用。 module.exports 初始值为一个空对象 {}，所以 exports 初始值也是 {}。 require() 返回的是 module.exports 而不是 exports。 eg：

  var name = 'nswbmw';
  exports.name = name;
  exports.sayName = function() {
    console.log(name);
  }

等价于：

  var name = 'nswbmw';
  module.exports.name = name;
  module.exports.sayName = function() {
    console.log(name);
  }

4：

var a = require("a"); // Mostly safe.
var foo = require("a").foo; // Dangerous!

exports.good = function (arg) {
  return a.foo("good", arg); // Uses the latest value of a.foo.
};

exports.bad = function (arg) {
  return foo("bad", arg); // Uses a stale value.
};
