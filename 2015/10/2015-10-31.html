HOW Z-INDEX WORKS

Frontend

HTML文档中元素在三维空间中生成,除了x-y轴,元素可以在z轴,第三维度。

z - index属性指定了两件事: 当前元素的叠加程度:属性只适用于定位元素。即有相对的位置,绝对或固定。有三个可能的值:

z-index: auto | <integer> | inherit

除了指定z-index,堆叠的一个元素是由许多因素控制的。元素是按照以下顺序堆放。

Position Description CSS 1 (bottom) The element forming the stacking context z-index: <integer> 2 Child elements with negative stack levels z-index: <negative integer> position: relative | absolute | fixed 3 In-flow, non-inline, non-positioned child elements display: /* not inline */ position: static 4 Non-positioned floating child elements float: left | right position: static 5 In-flow inline, non-positioned child elements display: /* inline */ position: static 6 Child elements with stacking level of 0 z-index: auto | 0 position: relative | absolute | fixed 7 (top) Child elements with positive stack levels z-index: <positive integer> position: relative | absolute | fixed

叠加规则：

  • 默认的叠加上下文根元素
  • 建立一个新z - index属性

  • 元素不能堆放高于(或低于)父元素的叠加的水平
